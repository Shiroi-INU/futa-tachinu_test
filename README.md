# 風太、立ちぬ

スマホで遊べる、10秒のタイミングミニゲームです。  
レッサーパンダの風太が、もう一度立ち上がることを目指します。

## 遊び方

1. 「立ち上がる」を押してゲーム開始
2. 左右に動く青いバーが、中央の赤い応援ラインに重なった瞬間にタップ
3. 10秒以内に100pt到達でクリア
4. 失敗するたびにアイテムを1つ獲得
5. 4回目の挑戦では、普通にタップしていればクリアできます

## 仕様

- 1プレイ15秒
- スマホ縦画面最適化
- ドット絵風ビジュアル
- 外部ライブラリなし
- HTML / CSS / JavaScriptのみ
- GitHub Pagesでそのまま公開可能

## アイテム

| 挑戦 | 獲得アイテム | 効果 |
|---|---|---|
| 1回目失敗後 | 新聞記事 | 次回の獲得ポイントアップ |
| 2回目失敗後 | 謎のサプリ | さらに獲得ポイントアップ |
| 3回目失敗後 | みんなの応援 | 4回目でクリアしやすくなる |

## ポイント設計

| 挑戦 | PERFECT | GOOD | MISS | 上限 |
|---|---:|---:|---:|---:|
| 1回目 | +8 | +5 | +0 | 92pt |
| 2回目 | +12 | +8 | +0 | 96pt |
| 3回目 | +20 | +12 | +1 | 99pt |
| 4回目 | +30 | +18 | +3 | なし |

補足：4回目は「普通にタップしていれば必ずクリア」の仕様を満たすため、6タップ以上して100pt未満だった場合、終了時に「みんなの応援」ボーナスで100ptに到達します。

## GitHub Pagesで公開する方法

1. GitHubで新規リポジトリを作成
2. このフォルダ内の4ファイルをアップロード
3. リポジトリの `Settings` を開く
4. `Pages` を選択
5. `Build and deployment` の `Source` を `Deploy from a branch` にする
6. `Branch` を `main` / `/root` にして保存
7. 表示されたURLにアクセス

## ローカルで確認する方法

`index.html` をブラウザで開くだけで遊べます。

より本番に近い確認をしたい場合は、フォルダ内で簡易サーバーを立てます。

```bash
python3 -m http.server 8000
```

その後、ブラウザで以下にアクセスします。

```text
http://localhost:8000
```

## 調整しやすい箇所

`script.js` 冒頭の `configs` を編集すると、各回のポイントや難易度を調整できます。

```js
{
  attempt: 1,
  perfect: 8,
  good: 5,
  miss: 0,
  cap: 92,
  perfectWindow: 12,
  goodWindow: 42,
  speed: 1.18
}
```

- `perfect / good / miss`：判定ごとの獲得ポイント
- `cap`：その回の最大ポイント
- `perfectWindow`：PERFECT判定の幅
- `goodWindow`：GOOD判定の幅
- `speed`：バーの移動速度
